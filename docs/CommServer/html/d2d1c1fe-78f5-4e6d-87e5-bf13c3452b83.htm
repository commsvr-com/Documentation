Introduction

This chapter describes how to prepare or tune a configuration for CommServer; it focuses on time settings and dependencies between them at different levels of configuration (mainly segment, port and data group).

Time settings for DataProviders

DataProvider configuration is the first place where time configuration appears in CommServer. Each DataProvider provides own settings; please find more information about those settings in a particular DataProvider manual:

  • MBUS configuration

  • MODBUS IP configuration

  • MODBUS RS configuration

  • SBUS IP configuration

  • SBUS RS configuration

Time settings for Segments nad Data Groups

Segments are the next place where time settings are essential. Communication segments are virtual groups of plant-floor devices that have the same address (IP, dial number, etc.) and need to be served using the same communication path. The basic scanning rule for the segment is that only one segment per channel is active at any instant to provide mutually exclusive access to common resources or communication medium. The segment is also responsible for connection management of connection oriented transport protocols, e.g. TCP.

One of the most important things to understand is a correlation between time settings for a segment and data group that is read through this segment (data groups are sets of process variables for which common scanning rules are defined). This is true for every configuration, from a very simple to the most advanced ones.

This section focuses on this issue. Please start reading from chapters about settings for Segments and Data Groups the and then follow the recommendations based on the following examples (none of the examples excludes the others):

  • The simplest approach (one station on one segment)

  • Two segments on one channel

  • Two stations on one segment, both on one channel
