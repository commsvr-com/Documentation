Diagnostics of time settings for segments and data groups

Diagnostic information about segment time settings are provided by “Server Monitor” on “Segment” tab or in CommServer Report. Especially, the report is very useful for diagnostic and analysis purposes. The figure below shows a part of the report:

Figure 1: Sample CommServer report (segments section)

When an analysis of time configuration is concerned the following elements are most interesting:

  1. Sampling time [ms] (from config) should be equal or very similar to Average sampling time [ms] (from real) (Sampling time is the period of time between connections (in milliseconds). Any configuration that produces the sample report from Figure 1 passes this condition.

  2. Time keep connect [ms] (from config) + Time idle keep connect [ms] (from config) should be similar to average Connect Time [ms]. Any configuration that produces the sample report from Figure 1 does not pass this condition; this means that segments remain connected for a longer period of time (around 1 second) than it was configured (Time keep connect + Time idle keep connect = 200 ms).

  3. The value of Data overtime should be very low (perfectly equal to 0). Any configuration that produces the sample report from Figure 1 does not pass this condition (see an explanation below to understand what this coefficient means).
