Version 2.00.12

Friday, May 19, 2011

This is only a bugfix release. The main changes are listed below:

  • Resolved issue with license installation (information about "wrong hardware key");

  • Resolved issue with displaying information about license failure.

Version 2.00.10

Monday, April 4, 2011

  • Some improvements to the user interface

  • Improved licensing management

  • Improved support for Windows Vista, 7 and 2008

  • New location (based on ALLUSERSPROFILE) of configuration and log files (see product manual for details

  • New version of OPC Viewer 3.10.00 is included

  • Support of x64 operating systems(DataPorter works as 32-bit application on x64 OS)

  • Small bugfixes and improvements

Version 2.00.08

  • New, extended help file is added

  • Some improvements and bug fixes

Thursday, May 31, 2010

  • Saving data in the database is fixed

  • Displaying graphic blocks in transactions configuration is fixed

Version 2.00.04

Thursday, May 13, 2010

  • Buffered queues are fixed

  • Some log messages and descriptions of paramteres are improved or fixed

Version 2.00.02

Friday, October 30, 2009

  • Tested for Compliance and Interoperability at OPC Interoperability Workshop (2009) in Nuremberg, Germany

  • Changes and improvements in user interface:

    • link to the license agreement

    • graphical improvements of DataProter Installer

    • sorting of the item list in DataPorter and OPC Viewer

  • OPC Viewer rel. 3.00.02 is a component of DataPorter 2.00.02

  • Added OPC Core Components as prerequisites for installation

  • Some improvements and bug fixes

Version 1.30.00

Wednesday, August 12, 2009

  • DataPorter has now the ability to run as Windows service. Nobody has to be logged in to run DataPorter. DataPorter as ordinary service can be configured to start automatically, run on a particular account, no interaction with a desktop is required

  • New DataPorter monitor is available. This new tool allows displaying the statistics from the DataPorter engine (from a local or remote PC):

    • The user interface in comparison to the previous DataPorter window is improved

    • New icons are used

    • New Toolstrip to control (start, stop restart) the local DataPorter service is added

    • New statistics information

    • Transaction statistics allows for an easier analysis and are more informative

  • Software is linked to new online DataPorter documentation.

  • New operation blocks are available: "DB Read" and "DB +". Those new blocks allow reading outputs from database operations. Those new blocks are preconfigured to proceed example activity

  • DataPorter configuration application (OPC Viewer) has a new toolstrip to control (start, stop restart) the local DataPorter service)

Version 1.21.01

Wednesday, January 07, 2009

  • Writing values from OPC to database is available in DataPorter demo version now. DataPorter can write data to any database. Built-in drivers allow for the connection to databases like Microsoft SQL, Oracle or any other using ODBC.

  • DataPorter installs preconfigured MsSQL database by default. It is very easy to start writing to that database; the only thing that must be done is adding the “DB save” block to transaction in the configuration (no programming, SQL scripting, or connection defining is necessary).

  • DataPorter allows writing to OPC from web services (previously, only reading from OPC via web services was available, now you can set the value in OPC using web services).

  • There is a new possibility of triggering the transactions. For any transaction a trigger item can be defined. A change of the value of this item executes the transaction. The user can define the change percentage that causes the execution

Version 1.2.0

Wednesday, September 03, 2008

  • New graphical user interface fully compatible with OPC Viewer (free OPC client from CommServer Software family). Thanks to new interface, browsing the server (off-line / on-line) and adding OPC items to configuration is straightforward.

  • Graphical transaction editor that supports creation of transactions as a diagram of operations. Each operation is presented as a simple block that can be easy connected with other operations.

  • New transaction engine supports more complex transactions. New way of displaying the transaction as tree diagram simplifies the configuration and makes it more readable.

  • New operations - Data Porter supports more mathematical functions; now it is possible to create a transaction that represents almost any mathematical equation.

  • New OPC client engine that supports synchronous and asynchronous communication and has more flexible configuration opportunities. It is easier now to optimize the connection.
