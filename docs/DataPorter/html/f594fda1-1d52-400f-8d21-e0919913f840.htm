Testing DataPorter Functionality

Home

In this section you learn how to prepare a testing environment for DataPorter in a few minutes only.

To prepare the testing environment you need to:

  • Download and install DataPorter

  • Download and install CommServer

  • Run DataPorter

 Note

When you install both DataPorter and CommServer on one PC you don’t need to configure DCOM.

Below, you can read where the CommServer installation package can be found and see small tutorials that allow you for fast end easy preparation of the testing environment.

CommServer download and configuration

For testing DataPorter functionality, download and install CommServer (OPC Server):

CommServer software download

CommServer is a redundant, multi-protocol, multi-channel OPC server for large scale distributed Systems. DataPorter and CommServer configurations were prepared for easy functionality testing. You only need to install CommServer to have a fully configured OPC server. If you want to create your own testing environment you can download short tutorials about CommServer configuration and using:

  • CommServer and SAIA: how to establish communication (SBUS Data Mode over RS)

  • CommServer Multi-Paths Redundancy (MPR)

  • CommServer Adaptive Sampling Algorithm (ASA)

When you have CommServer and DataPorter installed with default configuration, you need only to run DataPorter (see [Using DataPorter]). You will be able to see how DataPorter works. Test how easy you can create own transactions (See [ExampleTransaction ])

This documentation is designed to be used with CAS DataPorter (this software is available at http://www.commsvr.com).
