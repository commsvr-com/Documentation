This section describes the data bindings panel properties and data source configuration properties.

Data bindings properties

Below – the data bindings panel with a description of the particular properties.

Figure 1: Sample data binding panel and data source configuration window

Properties:

  1. General, vendor specific description of the bindings between this instance node in the model and underlying real-time process data source. The configuration data depends on the data provider plug-in that has been used.

  2. Redundant data sources collection - it provides an editor to add, remove or modify available data sources.

  3. Data binding description.

  4. Indication wheter the node is an instance declaration.

  5. Unique name of the node.

  6. Node class of this node.

  7. Name of the data source for the node

  8. Full name of the data binding (it is created automatically).

  9. Selection of the data source. Using a default plug-in for CommServer there are 3 possible data sources: CommServer (OPC server), OPC client (saved OPC Viewer session) and Simulator (random data generator).

  10. Button, which opens the window with a real-time process data source.

  11. Selection of the particular real-time process data source for the node.

  12. Name of the chosen data source.

This documentation is designed to be used with CAS CommServer (this software is available at http://www.commsvr.com).
