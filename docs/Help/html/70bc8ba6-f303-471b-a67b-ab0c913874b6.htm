Recommendations for time settings for Ports and Segments

Time settings for Ports are especially important when configuration of multi-path redundancy is required. In such case a particular station has two Ports on two independent Segments. Such situation is presented on the figure below (Figure 1).

Figure 1: Redundancy - One Station with two Ports and Segments

In such case Segments and Ports are switched continuously (according to the configuration). For an inactive port InactTime is counting down to 0 and thereafter the port changes (the active port becomes inactive and inactive port becomes active). It is important that Segments and Ports are switched independently and separately, so data transfer with a particular Station is possible only when an appropriate Port on Segment is Active.

The figure above presents an ideal situation when no failure occurs. But what if any failure occurs? Such situation is presented on the figure below.

Figure 2: Redundancy - transfer failure

When data transfer failure occurs and redundancy is configured properly, the port is switched immediately and data transfer is available. The next attempt to switch the port is after inactive time after failure (InactTimeAFailure) has elapsed. If there are no problems the inactive Port will be activated.

To learn more about configuration of redundancy it is recommended to:

  • Analyze the case delivered with a demo configuration

  • Watch a movie where a simple redundant configuration is presented ( available for download and on YouTube: Part 1 and Part 2 )
