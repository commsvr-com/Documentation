Introduction

This topic describes the properties of the OPC UA Address Space Model Designer solution.

Address Space Model Designer solution properties

Solution properties appear in the right panel after selecting a Solution node on a Model tree view. The description of the particular properties is shown below:

Figure 1: Address Space Model Designer solution properties

Description of the properties:

  1. Indication of the configuration plug-in of a UA server. This plug-in is used to configure the bindings of instance nodes in the model with the actual real-time process data sources. The configuration of the bindings can be edited using the “Process bindings” property available for all instance nodes.

  2. Display of a dialog to select a plug-in assembly that provides the configuration editor. This assembly is to be used to configure the UA server and all data sources available.

  3. Display of a dialog to select a file, that contains the UA server configuration.

  4. Path to the UA server configuration file.

  5. Information identifying the plug-in component.

  6. A text containing the assembly code base, i.e. the location of the assembly as specified originally, for example, in a System.Reflection.AssemblyName object.

  7. Name of the plug-in vendor.

  8. Information about plug-in copyright.

  9. Date and time of the last modification (release) of the plug-in component.

  10. Plug-in assembly description.

  11. Full name of the assembly.

  12. Unique identifier of the plug-in assembly.

  13. Plug-in assembly title.

  14. Plug-in assembly trademark.

  15. Plug-in assembly version.

  16. Home directory of the currently open solution.

  17. Solution name.
