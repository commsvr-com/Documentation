Custom configuration of the CommServer UA

Preparation of custom configuration for CommServer UA requires actions in different set of features accordingly to needs:

  • Common parts and functionalities of CommServer UA (read also: How to configure CommServerUA )

  • Modeling features of CommServer UA and OPC UA Address Space Model Designer – and Information Model must be prepared and built using OPC UA Address Space Model Designer tool.

  • Redundant, multi-protocol, multi-channel data access engine functionality – this configuration is required if process data access is necessary. In such case CommServer UA is configured in the same way as CommServer (DCOM/Classic), however additional step ( Process DataBinding ) is required.

  • o OPC wrapper functionality – this configuration is required if OPC DA (DCOM/Classic) data access is necessary. In such case CommServer UA is configured similar to DataPorter and usage of OPC Viewer tool is required to prepare such configuration. Read more in article: Walkthrough: Create UA Wrapper.
