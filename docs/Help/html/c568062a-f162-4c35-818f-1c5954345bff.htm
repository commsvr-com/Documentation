Environment monitoring

In a real environment, monitoring and management of the recourses making up the information and communication infrastructure are often of the same importance as access to real time data. We can distinguish the following types of information for this infrastructure:

  • the server software behavioral performance counters;

  • hosting hardware and operating system performance counters;

  • communication resources parameters and performance counters.

To address these needs, CommServer has built-in miscellaneous counters published on the OPC interface side as tags. These tags are updated in real time and can be used for diagnostic purposes. Similarly, each of the data providers can update counters predefined and published as OPC tags.

A specialized data provider called Monitor (MON) can be used to access hardware and operating system performance counters. Using this plug-in provided by the executing platform counters can be freely published as OPC tags. The counters can also be accessed remotely allowing to monitor other nodes of the process management system.

Using the plug-in concept it is possible to sample data from active devices distributed on the communication network and enable access to them in the standard way using OPC tags. According to the needs the data provider uses distributed agents collecting data locally and feeding them to the provider using proprietary protocols.

 Note

CommServer allows to access real data from the plant-floor and infrastructure behavioral data in a consistent way and integrate it into an existing business systems.
