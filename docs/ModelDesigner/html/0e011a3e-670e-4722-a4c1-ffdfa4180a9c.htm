Introduction

This topic describes the graphical notation that is used in OPC UA specifications and Address Space Model Designer (ASMD).

Notation

Before starting the modeling it is necessary to understand the graphical notation used by OPC UA and ASMD.

The OPC UA specification defines a graphical notation for OPC Unified Architecture Address Space data. It defines symbols that represent nodes classes from OPC UA. The table shows all the symbols from the specification and compares them with icons that are used by CAS Address Space Model Designer:

  • Object as a rectangle

  • ObjectType as a shadowed rectangle

  • Variable as a rectangle with rounded corners

  • VariableType as a shadowed rectangle with rounded corners

  • DataType as a shadowed hexagon

  • ReferenceType as a shadowed six-sided polygon

  • Method as an oval

  • View as a trapezium

Figure 1: Graphical notation

OPC UA graphical symbols are very similar to Model Designer icons.
