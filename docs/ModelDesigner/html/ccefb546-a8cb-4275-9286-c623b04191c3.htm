UA Server configuration

Below – a description of a property grid with OPC UA server configuration:

Figure 1: UA server configuration properties

Description of the server configuration properties:

  1. Vendor specific extension of the default UA configuration. For CommServer UA this extension is called "CAS extension". This extension contains the configuration of server specific functions.

  2. Configuration properties of the CommServer engine.

  3. Location of the CommServer engine configuration file (the default name is DefaultConfig.xml).

  4. Status of the CommServer engine configuration file (e.g. Opened).

  5. Configuration properties of namespaces, *.uanodes and *.csv files containing the configuration of the nodes.

  6. Configuration of the data bindings for particular nodes.

  7. Configuration properties of the OPC DA Client data source.

  8. Location of the OPC DA Client data source configuration file (OPC Viewer session file, default name for the SampleSolution is DemoConfiguration.oses).

  9. Status of the OPC DA Client data source configuration file (e.g. Opened).

  10. Configuration properties of the Simulator data source (random data generator).

  11. Descriptive name for the application (not necessarily unique).

  12. Type of the application. It might be Server, Client, ClientAndServer or DiscoveryServer.

  13. Unique identifier for the application instance.

  14. Mechanis, that validates the certificate.

  15. Additional configuration for client applications.

  16. Disabling / enabling of a high resolution clock.

  17. Additional configuration for discovery servers applications.

  18. Bucket to store additional application specific configuration data.

  19. ServiceMessageContext mechanism

  20. Unique identifier for the product.

  21. Security configuration for the application.

  22. Additional configuration for server applications.

  23. Path to the file that contains this configuration.

  24. Configuration of the trace and information about the log file.

  25. Collection that contains the transport configuration for the application.

  26. Quotas that are used at the transport layer.

  27. Indication whether the native stack should be used.

This documentation is designed to be used with CAS CommServer (this software is available at http://www.commsvr.com).
